## О проекте

это веб-ориентированная система для автоматизированного обнаружения, анализа и мониторинга уязвимостей информационной инфраструктуры. Проект разработан в рамках выпускной квалификационной работы (дипломного проекта) и демонстрирует современные подходы к управлению информационной безопасностью.

### Цель работы

Разработка прототипа системы, позволяющей:
- Автоматизировать процесс сбора информации об активах и открытых портах
- Сопоставлять версии сервисов с актуальными базами уязвимостей (CVE)
- Оценивать риски на основе международного стандарта CVSS
- Формировать аналитические отчеты для принятия обоснованных решений

### Актуальность

В условиях растущего числа кибератак и усложнения IT-инфраструктуры, ручной анализ уязвимостей становится неэффективным. Система предоставляет инструментарий для автоматизации этого процесса, позволяя специалистам по информационной безопасности фокусироваться на критических уязвимостях.

## Функциональные возможности

### 1. Авторизация и аутентификация
- Регистрация и вход в систему
- JWT-токены для защиты API
- Ролевая модель (администратор, пользователь)

### 2. Сканирование активов
- Автоматическое обнаружение хостов в сети
- Сканирование портов (TCP/UDP)
- Идентификация версий сервисов (banner grabbing)

### 3. Управление активами
- Добавление/редактирование/удаление активов
- История изменений (кто и когда менял)
- Группировка по проектам/отделам

### 4. Анализ уязвимостей
- Сопоставление версий сервисов с базой CVE
- Автоматический расчет CVSS v3.1
- Классификация по уровню критичности (Critical/High/Medium/Low)

### 5. Дашборды и аналитика
- Распределение уязвимостей по критичности (pie chart)
- Статистика по активам
- Динамика появления/исправления уязвимостей (line chart)
- Тренды безопасности за период

### 6. Отчетность
- PDF-отчеты с полной информацией по активам и уязвимостям
- CSV-экспорт для интеграции с другими системами

### 7. Логирование
- Логи сканирований (статус, длительность, результаты)
- Логи активности пользователей (вход, действия, экспорты)

### 8. Управление пользователями
- Создание и блокировка учетных записей
- Назначение ролей и прав доступа

## Установка и запуск

### Предварительные требования
- Node.js (v18 или выше)
- PostgreSQL (v14 или выше)
- pnpm

### 1. Клонирование репозитория
```bash
git clone https://github.com/ksunwy/information-security-monitoring.git
```

### 2. Настройка окружения
```
cp .env.example .env
# Отредактируйте .env файл, указав параметры подключения к БД и секретные ключи
```
### 3. Установка зависимостей

```
# Установка зависимостей для backend
cd backend
pnpm install

# Установка зависимостей для frontend
cd frontend
pnpm install
```

### 4. Запуск приложения
```
Режим разработки:
# Терминал 1 - Backend
cd backend
pnpm run dev:start

# Терминал 2 - Frontend
cd frontend
pnpm run dev
```

### 5. Доступ к приложению

- Frontend: http://localhost:5173
- Backend API: http://localhost:3000/api


## Демонстрация
Основные сценарии использования
Сканирование нового актива

```
Активы → Добавить актив → Ввести IP/домен → Запустить сканирование
→ Дождаться завершения → Просмотреть найденные уязвимости
Анализ уязвимостей
```

```
Уязвимости → Выбрать актив → Изучить CVE → Оценить CVSS score → Экспортировать отчет
```
```
Дашборд → Выбрать период → Анализировать динамику → Принять меры
```
